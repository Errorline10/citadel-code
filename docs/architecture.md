# Architecture Notes

This document tracks implementation architecture for The Great Citadel of Knowledge.

## Current State

Architecture has not yet been selected.

## Initial Direction

The codebase should keep lore-derived data separate from application logic.

Possible early layers:

- data schemas
- room/map model
- object and inventory model
- ability model
- gate and puzzle model
- journal/log model
- interface layer

## Open Decisions

- First implementation target
- Runtime/platform
- Language/framework
- Data format conventions
